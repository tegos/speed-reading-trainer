# Speed Reading Trainer — vanilla rewrite design

**Дата:** 2026-06-06
**Статус:** затверджено
**Замінює:** React 19 версію (PR #1) — повний rewrite на vanilla TypeScript

## Мета

Пет-проект: тренажер швидкочитання, який реально корисний. Теплий «книжковий» UX,
нуль runtime-залежностей, дані локально.

## Рішення і обґрунтування

Дизайн спирається на дослідження читання (Rayner et al. 2016, *Psych. Science in the
Public Interest*; Schotter, Tran & Rayner 2014):

- Чанк-підсвітка — це **пейсер** (задає темп), а не «сприйняття груп слів»
  (перцептивний спан ~15 літер це виключає). Не обіцяємо магії.
- **Регресії потрібні для розуміння** → клік по слову = перемотка назад, завжди доступна.
- RSVP прибирає регресії → позиціонується як режим коротких темпових тренувань,
  з паузами на пунктуації та довгих словах.
- Понад ~500–600 wpm розуміння падає → слайдер swpm обмежений 60–600.

## Режими

Гібрид, перемикач у хедері. Спільний стан: текст, позиція (індекс слова), wpm.
Перемикання режиму зберігає позицію.

### Книга (основний)
- Текст → слова → `<span>`-и. Активний чанк (1–5 слів, слайдер) — ink-blue tag;
  прочитане — приглушений колір.
- Клік по слову → позиція переноситься туди (регресія/перемотка).
- Прогрес-бар + оцінка «~N хв лишилось» (за поточним wpm).
- Тік таймера: інтервал = `60000 / wpm * chunkSize` мс на чанк.

### RSVP
- Темна сцена (`#141413`), слово по центру, ORP-літера акцентом (`#2D5A8A`),
  вертикальні риски-гайди зверху/знизу.
- ORP: позиція літери ≈ 35% довжини слова (індекс `round((len-1)*0.35)`).
- Затримки: базова `60000/wpm` мс; ×2 після `.!?;:`, ×1.5 для слів 8+ літер,
  ×1.3 після коми.
- Лічильник «слово N з M», wpm поруч.

## Тренування

- **Авто-прогресія** (опційний перемикач): +10 wpm кожні 2 хв безперервного читання.
  Скидається на pause.
- **Сесія**: від play до pause або кінця тексту. Фіксуємо: дата, слова прочитані,
  хвилини, ефективний wpm. Сесії < 10 c не записуємо (шум).

## Дані (localStorage)

```ts
interface LibraryText { id: string; title: string; content: string; position: number; addedAt: number }
interface Settings { wpm: number; chunkSize: number; fontSize: number; mode: 'book'|'rsvp'; autoProgress: boolean; activeTextId: string|null }
interface Session { date: number; words: number; minutes: number; wpm: number }
```

Ключі: `srt:library`, `srt:settings`, `srt:sessions`. Версіонування: `srt:v = 1`.
`title` — перший рядок/речення тексту (обрізаний до ~60 символів).

## UI-каркас

- **Хедер**: назва, перемикач Книга/RSVP, кнопки Бібліотека та Статистика.
- **Читалка**: центр, `max-width: 68ch`.
- **Нижня панель**: play/pause, wpm-слайдер (60–600), чанк (1–5), шрифт (14–24),
  перемикач авто-прогресії.
- **Бібліотека** (оверлей): список текстів з прогресом кожного, видалення, вибір активного.
- **Статистика** (оверлей): таблиця останніх сесій + SVG-графік wpm за датами (без бібліотек).
- **Завантаження тексту**: Ctrl+V (paste), Ctrl+O (файл .txt), drag&drop файлу на вікно.
  Новий текст додається в бібліотеку і стає активним.
- **Клавіші**: пробіл — play/pause, ←/→ — ±25 wpm, F5 — скинути позицію.

## Архітектура

Vite + TypeScript, нуль runtime-залежностей. Модулі + міні-store (підхід A):

```
src/
  store.ts        — стан + subscribe (pub/sub, один setState)
  engine.ts       — таймер, позиція, чанки, авто-прогресія, межі сесій
  persist.ts      — localStorage: load/save, версіонування
  modes/book.ts   — рендер книжкового режиму, клік-перемотка
  modes/rsvp.ts   — рендер RSVP-сцени, ORP, затримки
  panels/library.ts
  panels/stats.ts — таблиця + SVG-графік
  ui/toolbar.ts   — нижня панель, хедер, гарячі клавіші
  main.ts         — композиція
```

- `engine.ts` — чиста логіка без DOM: тестується Vitest.
- `persist.ts` — тестується з mock localStorage.
- Рендер-модулі: функції `mount(el, store)`, підписка на store.

## Обробка помилок

- Не-текстовий файл / порожній paste → недіалогове повідомлення (toast на parchment).
- localStorage переповнений (великі тексти) → повідомлення, текст не зберігається
  в бібліотеку, але читається з пам'яті.
- Збій парсингу збережених даних → скидання до дефолтів (не падати).

## Тестування

- Vitest: `engine` (чанки, прогресія, межі сесій, затримки RSVP), `persist`
  (round-trip, версія, биті дані).
- Рендер — ручний smoke через `npm run dev` (пет-проект, без e2e).

## Поза скоупом

- Синхронізація між пристроями, акаунти, PWA/offline.
